# Trae 博客写作指南

> 写给 Trae（或其他 AI 编辑器）：当需要生成或修改博客文章时，请遵循以下规则。

## 一、文章放在哪里

| 目录 | 说明 |
|------|------|
| `content/posts/` | **最新文章放这里**（Trae Solo CN 生成） |
| `posts/` | 旧文章目录，一般不动 |

文章是 `.md` 文件，UTF-8 编码。

## 二、Front Matter（文章头部）

每篇文章必须以 `---` 包裹的 YAML 头部开头：

```yaml
---
title: "文章标题"
date: 2026-05-18
draft: false
tags: ["标签1", "标签2"]
summary: "文章摘要，会显示在卡片上"
---
```

> ⚠️ `tags` 必须是 YAML 数组格式，不要用逗号分隔的字符串。

## 三、正文只用 Markdown

博客的 `build.js` 会将 Markdown 转换为 HTML，然后注入 JavaScript 模板字符串。

**支持的 Markdown 语法：**

```markdown
# 一级标题（不要用）
## 二级标题（文章主标题）
### 三级标题（小节标题）

**粗体文本**
*斜体文本*

> 引用块

- 列表项1
- 列表项2

`行内代码`

```代码块（三个反引号）

[链接文字](https://example.com)

![图片说明](/images/图片.jpg)
```

## 四、🚫 绝对禁止

### 1. 不要在文章里写 `<style>`

```html
<style>
  /* ❌ 绝对不行！这会破坏深色模式 */
  .my-class { background: #fff; color: #555; }
</style>
```

**替代方案：** 用纯 Markdown，不要自定义样式。

### 2. 不要硬编码颜色

| ❌ 错误写法 | ✅ 正确做法 |
|-------------|-------------|
| `color: #333` | 用纯文本，不用指定颜色 |
| `background: #fff` | 别写，让主题控制 |
| `color: white` | 同上 |

博客是**深色主题**，硬编码浅色会让文字看不见。

### 3. 不要写裸 `</script>`

```
<!-- ❌ 这会导致 build.js 出错 -->
</script>

<!-- ✅ build.js 会自动转义，但最好避免 -->
```

### 4. 不要自定义 CSS 类名

文章内容最终渲染在 `.d-content` 容器内，所有样式由博客主题统一控制。

## 五、文章的渲染流程

```
你的 .md 文件
    ↓
build.js 读取并解析
    ↓
Markdown → HTML 转换
    ↓
注入 index.html 的 JavaScript 数组
    ↓
浏览器渲染到 .d-content 容器内
    ↓
CSS 变量控制所有样式（深色模式）
```

## 六、可用的主题色（供参考）

博客使用 CSS 变量控制颜色，所有文字、背景都通过以下变量定义：

```
--bg:  #0a0a0f   (最深背景)
--bg2: #12121a   (卡片背景)
--bg3: #1a1a25   (强调背景/代码块)
--bdr: #2a2a3a   (边框)
--tx:  #eee      (主文字)
--tx2: #aaa      (次要文字)
--tx3: #666      (最淡文字)
--ac:  #00e89d   (主题色/绿色)
--ac2: rgba(0,232,157,.12)  (主题色透明)
```

## 七、文章内容建议

- **标题层级**：用 `##` 和 `###`，不要用 `#`
- **段落间距**：空一行就是新段落
- **代码块**：用三个反引号包裹，可以指定语言
- **图片**：放在仓库 `images/` 目录，用相对路径引用
- **列表**：用 `-` 前缀，嵌套列表缩进 2 空格
- **不要使用**：表格（样式不统一）、自定义 div/span、内联样式

## 八、写完后

1. 把 `.md` 文件放到 `content/posts/`
2. 提交 GitHub → blog-poller 自动构建部署
3. 约 2 分钟后刷新博客就能看到

---

**一句话总结：纯 Markdown，别写样式，让主题管样式。**
