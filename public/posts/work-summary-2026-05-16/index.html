    🚀 今天做了什么#

    一个下午 + 晚上，用 OpenClaw 从零搭建了两个网站：

    • 个人介绍站：https://yadongw.dpdns.org
    • 博客站：https://yadongw118.dpdns.org

    🌸 个人介绍站：3D 樱花浮岛#

    这是最花时间的部分。最终实现了一个带有电影感的互动网页：

    核心功能#

    3D 樱花浮岛

    • 使用 Three.js r128 构建低多边形浮岛
    • 三层结构：草地 + 泥土 + 石头
    • 樱花树带粒子花瓣飘落效果
    • 萤火虫 + 星空背景

    互动系统

    • 点击树枝：花瓣爆发 + 对白气泡（“你也来看樱花吗？"、“风会记得花的方向”…）
    • 点击月亮：月相变化（满月→下弦→新月→上弦循环）+ 粉色文字
    • 鼠标移动：风向跟随，花瓣飘动方向改变
    • 全局风场：自动波动，像电影镜头

    氛围层

    • 月光光晕（呼吸动画）
    • 底部雾气（双层飘动）
    • 场景明暗呼吸效果
    • 暗角效果

    音乐播放器

    • 两首歌可切换：告白气球、迷失幻境
    • 点击页面任意位置触发播放
    • 5 秒后自动隐藏，点击重新显示

    其他

    • 深浅色模式切换
    • 右侧滚动进度条（可拖拽）
    • 响应式适配（手机/平板/桌面）
    • 照片压缩优化（8.9MB → 55KB）

    📝 博客站：Hugo + 匿名评论#

    搭建过程#

    1. 克隆 GitHub 仓库（2326239842/my-bok）
    2. 配置 Hugo 主题（PaperMod）
    3. 解决 Hugo 兼容性问题（升级到 v0.152）
    4. 配置 raw HTML 渲染（支持文章内嵌 CSS/JS）
    5. 部署到 Cloudflare Pages

    评论系统#

    这是个有意思的挑战。最初尝试了 Twikoo（需要后端服务器）和 Vercel（需要手机验证），最后选择了自建方案：

    技术方案：Cloudflare Pages Functions + KV

    用户评论 → Pages Function API → KV 存储 → 返回评论列表

    实现细节：

    • Worker 处理 GET/POST 请求
    • KV 存储评论数据（按文章 slug 分组）
    • 完全匿名，名字自定义
    • 所有访客共享评论
    • 零费用

    效果：

    • 任何人可评论
    • 不需要注册/登录
    • 评论实时显示
    • 支持所有文章

    🔧 技术栈总结#

    组件技术
    3D 渲染Three.js r128
    樱花互动Canvas 2D + 粒子系统
    评论后端Cloudflare Workers
    数据存储Cloudflare KV
    博客框架Hugo + PaperMod
    部署平台Cloudflare Pages
    图片压缩Python PIL
    版本管理GitHub + Git

    💡 经验教训#

    1. Three.js CDN 在国内可能加载慢，但这次测试没问题
    2. HEIC 格式图片需要转换才能用在网页上
    3. Cloudflare Workers 部署简单，但自定义域名需要配置路由
    4. Hugo 的 raw HTML 默认会被过滤，需要配置 goldmark.renderer.unsafe = true
    5. 移动端触摸事件需要仔细处理，否则会阻止页面滚动
