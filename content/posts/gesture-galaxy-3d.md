## 这是什么

这是一个跑在浏览器里的 **3D粒子星系**，用摄像头追踪你的手部动作来控制星系的旋转、缩放和形态切换。

不需要安装任何插件，不需要下载任何APP，打开网页就能用。

核心就两个东西：

- **Three.js** — 负责渲染14000个粒子组成的3D场景
- **MediaPipe Hands** — 负责实时追踪手部21个关键点

## 手势控制

如果浏览器有摄像头权限，你可以用手势来操控：

- **✋ 移动手掌** → 旋转星系视角
- **✊ 握拳/张开** → 缩放
- **🤏 捏合手指** → 切换到下一种形态
- **🤲 双手靠近** → 触发爱心形态

没有摄像头也没关系，鼠标拖拽可以旋转，滚轮可以缩放，底部按钮可以切换形态。

MediaPipe 的模型文件全部自托管在国内 CDN 上（不用走 Google 的服务器），加载速度快，不翻墙也能用。

## 技术实现

### 粒子系统

用 Three.js 的 `BufferGeometry` 创建了 14000 个粒子，每个粒子有独立的位置、颜色和大小。不同形态有不同的生成函数，比如星系形态用对数螺线方程：

```
r = a * e^(b * θ)
```

粒子沿旋臂分布，加上随机偏移模拟真实星系的弥散感。

### 手部追踪

MediaPipe Hands 提供 21 个手部关键点的实时坐标。我只用了其中几个关键点来做手势识别：

- 拇指尖和食指尖的距离 → 判断捏合
- 手掌中心位置 → 控制旋转
- 手掌开合程度 → 控制缩放

### 双手交互

支持同时追踪两只手。当两只手靠近时，触发特殊形态（爱心）。左手和右手各自独立控制不同的交互维度。

### 性能优化

14000个粒子每一帧都要更新位置，性能是关键：

- 用 `BufferGeometry` 而不是 `Geometry`，直接操作 Float32Array
- 形态切换时用 lerp 插值，不用 tween 库
- MediaPipe 模型文件自托管 + preload，减少加载时间
- `.tflite` 文件重命名为 `.bin` 避免 MIME 类型问题

## 部署

整个项目是一个纯静态页面，部署在 Cloudflare Pages 上。MediaPipe 的 WASM 文件、Three.js、手部追踪模型全部自托管，不依赖任何外部 CDN。

源文件结构：

```
galaxy/
├── index.html          # 主页面（HTML + CSS + JS 全在一个文件）
└── mediapipe/
    ├── hands.js        # MediaPipe Hands API
    ├── *.wasm          # WASM 运行时
    ├── *.bin           # 手部追踪模型
    └── three.min.js    # Three.js
```

没有构建工具，没有 npm，没有打包——一个 HTML 文件加一堆静态资源，拖到 Cloudflare Pages 就上线了。
